# 제시 단어 기반 카드뉴스 생성 프로젝트

## 한 줄 컨셉
- 사용자가 아무 단어나 주면, AI가 해당 키워드와 연결된 **최신·핫 이슈를 요약해 2~3장 카드뉴스**로 자동 생성한다.

## 문제 정의
- 뉴스는 너무 길고, 한 번에 핵심 파악이 어렵다.
- 사용자는 관심 키워드 중심으로 빠르게 트렌드를 보고 싶다.
- 기존 요약 서비스는 범용 요약이 많고, **"내가 준 단어" 중심 맥락화**가 약하다.

## 목표
- 입력 키워드 1개로, 1분 내에 소비 가능한 카드뉴스 생성
- 카드 2~3장에 핵심 맥락(무슨 일/왜 중요한지/지금 시점 포인트) 전달
- 신뢰도 확보를 위해 출처/시간 정보 제공

## 타깃 사용자
- 바쁜 직장인 (출근/점심 시간에 빠르게 트렌드 체크)
- 콘텐츠 기획자/마케터 (아이디어 발굴)
- 일반 사용자 (관심사 키워드 기반 요약)

## 핵심 기능 (MVP)
1. **키워드 입력**
    - 예: "엔비디아", "총선", "아이폰", "금리", "서울 집값"
2. **최신 이슈 수집**
    - 최근 24~72시간 내 기사/공식 자료 우선
3. **핵심 이슈 2~3개 클러스터링**
    - 같은 이슈는 묶고, 중복 제거
4. **카드뉴스 생성 (2~3장)**
    - 카드1: 한 줄 요약 + 배경
    - 카드2: 지금 왜 뜨는지(핵심 데이터/발언)
    - 카드3: 앞으로 볼 포인트(영향/체크포인트)
5. **출처 표시**
    - 카드 하단에 매체명 + 발행 시각 + 링크

## 카드 템플릿 예시
### 카드 1 — 무슨 이슈?
- 제목: `[키워드] 지금 무슨 일?`
- 본문: 최근 핵심 사건 1~2줄 요약

### 카드 2 — 왜 중요해?
- 제목: `왜 지금 뜨는가`
- 본문: 숫자/발언/정책 변화 등 핵심 근거 2~3개

### 카드 3 — 앞으로 관전 포인트
- 제목: `다음 체크포인트`
- 본문: 향후 일정, 리스크, 수혜/피해 가능성

## 품질 기준
- 최신성: 72시간 이내 소스 우선
- 신뢰성: 확인 가능한 출처 최소 2개 이상
- 중립성: 과장/선동 표현 금지
- 간결성: 카드 1장당 120~180자 내외

## 기술 구성 초안
- 수집: 검색 API/뉴스 피드 + 본문 fetch
- 정제: 중복 제거, 키워드 연관도 스코어링
- 생성: LLM 요약 + 카드 구조화(JSON)
- 렌더링: 템플릿 기반 이미지 카드(PNG/WebP)
- 배포: 링크 공유 또는 텔레그램/웹앱 전송

## 데이터 플로우
1) 키워드 입력
2) 검색/수집
3) 기사 정제(중복 제거, 신뢰도 필터)
4) 이슈 클러스터링
5) 카드 문구 생성
6) 카드 렌더링
7) 사용자에게 2~3장 전달

## 리스크 & 대응
- 가짜뉴스/낚시성 기사 혼입 → 출처 화이트리스트 + 다중 출처 교차확인
- 정치/사회 이슈 편향 → 상반된 시각 최소 1개 포함 규칙
- 키워드 모호성("애플") → 입력 직후 분야 선택(기업/과일)
- 실시간성 저하 → 캐시 TTL 짧게 + 긴급 재수집 버튼

## 성공 지표 (KPI)
- 카드 생성 완료율
- 평균 생성 시간
- 재사용률(다시 검색/다른 키워드 입력)
- 공유율(링크 공유, 저장)
- 사용자 만족도(간단 thumbs up/down)

## 확장 아이디어
- 개인화: 관심 키워드 구독 + 아침 브리핑
- 톤 선택: 중립/쉽게/전문가형
- 다국어 카드 생성
- 카드 → 숏폼 스크립트 자동 변환
