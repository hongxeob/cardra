# Cardra 개발 로그 (2026-02-19)

## 1. 서비스 컨셉 피벗 (Pivot)
- **변경 전**: 아이디어를 카드뉴스로 변환하는 도구
- **변경 후**: 사용자의 키워드 입력에 따라 **최신·핫 이슈를 요약하여 2~3장 카드뉴스**로 자동 생성하는 서비스
- **이유**: 단순 변환보다 실시간 트렌드 요약이라는 명확한 가치 제안(Value Proposition)이 사용자 유입에 더 유리하다고 판단.

## 2. 디자인 시스템 및 UI/UX 고도화
### 컬러 및 타이포그래피
- **Main Color**: `#00A676` (Mint Green) - 신뢰감 있고 현대적인 뉴스 매체의 느낌.
- **Sub Color**: `#E0D0C1` (Beige Sand) - 따뜻하고 세련된 보조색으로 카드 배경 등에 활용.
- **Font**: `Pretendard Variable` - 가독성 최우선.

### 주요 구현 사항
- **Global Header**: 서비스 정체성(Cardra Logo)과 네비게이션을 위해 `backdrop-filter`가 적용된 고정 헤더 추가.
- **LocalStorage Persistence**: 서버 API 없이도 사용자가 생성한 카드를 다시 볼 수 있도록 브라우저 저장소 연동.
- **UX 애니메이션**: 페이지 전환 시 `fadeIn`, 카드 로딩 시 순차적 `slideUp` 효과를 주어 앱의 감도 향상.
- **데이터 시각화**: 리서치 결과의 신뢰도를 단순 수치가 아닌 CSS 게이지 바(Gauge Bar)로 시각화.

## 3. 기능 구현 및 서버 연동
- **카테고리 기반 추천**: 서버 DTO(`categoryId`)를 활용하여 IT, 건강, 경제 등 관심 분야별 키워드 추천 UI 구현.
- **딥 리서치(Deep Research) 결과 고도화**: 
  - 기존 JSON 출력을 전문적인 **AI 분석 리포트 뷰**로 전환.
  - 팩트체크 항목, 리스크 플래그, 상세 출처 카드를 포함하는 레이아웃 설계.
- **비동기 작업 상태 관리**: 리서치 Job의 상태(`completed`, `failed`)에 따라 로딩 스피너 색상과 상태 메시지가 동적으로 변화하도록 구현.

## 4. 트러블슈팅 및 리팩토링
- **Babel Syntax Error**: JSX 내 `<style>` 태그의 템플릿 리터럴 이스케이프 오류 수정.
- **Import Resolution Fix**: `src` 폴더 외부에 있던 `features/research` 폴더를 `src/features/research`로 이동하여 Vite 빌드 오류 해결.

## 5. 결과물 확인
- 모든 프론트엔드 변경 사항은 커밋 및 푸시 완료 (`feat(web): enhance UI/UX and integrate server-side features`).
