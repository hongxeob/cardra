# Cardra 콘텐츠 오케스트레이션 가이드

## 목표
수집 데이터 기반으로 카드뉴스 스크립트를 1~3장으로 일관되게 자동 생성.

## 파이프라인
1. 입력 정규화(기사/리포트/통계)
2. 핵심 메시지 추출(영향도, 최신성, 검증가능성)
3. 메시지 맵 구성(메인 1개 + 서브 최대 3개)
4. 카드 매핑
   - Card1: 맥락(왜 봐야 하는지)
   - Card2: 근거(수치/변화)
   - Card3: 의미(결론/다음 액션)
5. 톤/구조 검수 → 게시 가능안 산출

## 톤 규칙
- 명료성/중립성 우선
- 과장·선동 문구 제한(예: 무조건/폭락/반드시)
- 출처 없는 전망 단정 표현 금지

## 카드 형식(예시)
- 제목: 14~24자
- 요약: 2문장 이내, 약 90자
- 출처 라벨: `출처: 기관/매체, 날짜, 링크`

## 일관성 점검(체크리스트)
- 카드 간 메시지 충돌 없음
- 용어 표기 통일(예: YoY/전년동기)
- 카드당 핵심 포인트 1개
- 수치 단위 가독성(억/만/%)
- 핵심 수치에 근거 라벨 존재

## 구현 포인트(백엔드)
- `CardService` 입력은 `fact summary`를 받는 형태로 진화 가능
- 카드별 템플릿 정보(`tone`, `template`)를 DTO에 선택적 확장
- QA 실패 시 `partial` 응답 혹은 fallback 카드 메시지로 전환 가능
